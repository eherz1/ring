EventSystem
An event system implementation. EventSystem is a system that is primarily interested in responding to events that occur on the world to which the system belongs. It accepts a set of subjects mapped to callback functions, the subjects of which will be subscribed to automatically during initialization and unsubscribed from during destruction. The system may implement update and process functions as well, but these are considered optional.

EntitySystem
An entity system implementation. An entity system keeps a set of entity IDs and calls its updateEntity and processEntity functions iteratively for each entity ID in the set. Membership in the set is determined by a call to matchEntity on the following events, (1) "+#" aka entity created, (2) "-#" aka entity destroyed, (3) "+@" aka component added, (4) "-@" aka component removed.

Fields:

  • initialize ()=>void An optional function which will be called when the system is added to a world.
  • destroy ()=>void An optional function which will be called when the system is removed from a world.
  • updateEntity (dt,entityId)=>void A required function which performs updates before frame processing on all matching entities.
  • processEntity ()=>void A required function which performs per-frame processing on all matching entities.

World
A world implementation. World represents a context in which a set of entities with various compositional components are acted upon by a ordered set of of systems.

Fields:

  • subscribe (subject,callback)=>void Subscribe to a ring.SubjectString.
  • unsubscribe (subject,callback)=>void Unsubscribe from a ring.SubjectString.
  • publish (subject,message)=>void Publish a message to a ring.SubjectString.
  • putComponent (entityId,componentName,value)=>void Add a component to an entity. The form of a component is unstructured, but if values are provided, the component will be initialized with those values. Otherwise, the component will be initialized to boolean 'true'. Aliases: putc,addComponent,addc.
  • deleteComponent (entityId,componentName)=>void Delete a component from an entity. Aliases: delc,removeComponent,remc.
  • getComponent (entityId,componentName,[componentKey])=>void Get a copy of a component value or a copy of the entire component table. Aliases: getc.
  • hasComponent (entityId,componentName)=>boolean Check whether a entity has a component. Aliases: hasc.
  • createEntity ({componentName,defaults},...},[name])=>void Create an entity.
  • getEntity (name)=>void Get an entity by name.
  • destroyEntity (entityId)=>void Destroy an entity.
  • addSystem (system,[name])=>void Add a system to this world.
  • getSystem (name)=>void Get a system from this world.
  • removeSystem (systemOrName)=>void Remove a system from this world.
  • update (dt)=>void Perform world updates.
  • process ()=>void Perform world per-frame processing.
